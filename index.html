권한(permissions) 규칙 설정

도구 사용 범위를 허용(allow) 또는 차단(deny) 규칙으로 직접 제어할 수 있습니다. 차단 규칙이 항상 먼저 적용됩니다. (평가 순서: deny → ask → allow)

규칙 형식

  • Tool : 도구 전체에 적용
  • Tool(specifier) : 특정 조건만 적용

작성 예시

  • Bash(npm run *) : npm run으로 시작하는 명령만 허용
  • Read(./.env) : .env 파일 읽기 차단
  • Bash(rm -rf *) : rm -rf 명령 차단

작업 제어

Foreground / Background 모드

Foreground (일반 실행)

  • Claude가 명령 완료를 얌전히 기다림
  • 작업이 끝날 때까지 다른 프롬프트 입력 불가
  • 타임아웃(기본 2분) 초과 시 강제 종료됨

Background (백그라운드) 단축키: Ctrl + B

  • 명령을 비동기로 뒤에서 몰래 실행 (ID 반환)
  • 실행되는 동안 Claude에게 다른 작업 동시 요청 가능
  • npm install, docker build, 서버 띄우기 등에 유용

작업 목록 관리 (Tasks)

현재 백그라운드에서 돌고 있는 프로세스(최대 10개)를 확인하고 제어합니다.

  • 목록 조회 및 관리 /tasks
  • 작업 목록 뷰 하단 토글 Ctrl + T
  • 모든 백그라운드 강제 종료 Ctrl + F

타임아웃(Timeout) 방지

오래 걸리는 작업(빌드, 대용량 처리)이 Foreground에서 2분 만에 튕기는 것을 막는 방법입니다.

방법 1: 실행 중 Ctrl + B를 눌러 백그라운드로 전환
방법 2: settings.json 직접 수정
5시간(18000000ms)으로 변경 
settings.json 의 bash timeout을 18000000 (5시간) 으로 설정해줘

BASIC 05

Hooks

Hooks 란?

Claude Code가 이벤트를 감지하면 settings.json에 등록해 둔 내 스크립트를 자동으로 실행하는 기능입니다.

도구 실행 전 차단, 응답 완료 후 알림, 파일 수정 후 정리처럼 반복 작업을 자동화할 수 있습니다.

활용 예시
  • 위험 명령 차단 — rm -rf 실행 전 자동으로 막기
  • 작업 완료 알림 — 응답이 끝나면 PC 알림 띄우기
  • 코드 자동 정리 — 파일 수정 후 자동으로 linter 실행
Claude Code 전체 흐름에 18개 시점이 존재하며, 차단 또는 관찰 용도로 활용합니다

const c = j.tool_input && j.tool_input.command || '';
if (/\b(rm|rmdir|del|Remove-Item)\b/.test(c)) {
  console.log(JSON.stringify({
    hookEventName: 'PreToolUse',
    permissionDecision: 'deny',
    permissionDecisionReason: '삭제 명령어 차단: archive/이동 등 안전한 대안 사용'
  }));
  process.exit(0);
}

BASIC 07

기존 MCP vs Plugin

두 방식의 차이점을 비교합니다

기존 MCP (stdio)

  • git clone + npm build 필요
  • read-messages / send-message 2가지
  • 멘션 필터링 불가
  • 파일 첨부 / 이모지 반응 불가
  • DM 전용 (서버 채널 제한적)

Plugin (공식)

  • 자동 설치 — 빌드 과정 없음
  • reply / fetch / react / edit / download
  • 실시간 메시지 수신 (channel 태그)
  • 파일 첨부 / 이모지 반응 지원
  • DM + 서버 채널 (access.json)
Plugin = 공식 지원 + 더 많은 도구 + 실시간 수신 — MCP보다 간편하고 강력합니다

AI 에이전트 상세 이해

AI 에이전트는 쉽게 말하면 “목표를 받고, 스스로 판단해서 필요한 작업을 순서대로 실행하는 AI 시스템”입니다.
일반 챗봇이 “질문 → 답변” 중심이라면, AI 에이전트는 한 단계 더 나아가서 목표 달성을 위해 계획하고, 도구를 사용하고, 결과를 확인하면서 작업을 이어가는 구조라고 볼 수 있습니다.

1. AI 에이전트의 기본 작동 흐름

AI 에이전트는 보통 아래 순서로 작동합니다.

목표 입력 목표 이해 작업 계획 도구 선택 작업 실행 결과 확인 수정/반복 최종 결과

예를 들어 "이번 주 제주도 여행 계획 짜줘."라고 했을 때 일반 AI는 바로 코스를 추천하지만, AI 에이전트는 다릅니다.

  1. 여행 날짜 확인
  2. 날씨 검색
  3. 항공권/숙소 조건 확인
  4. 관광지 후보 정리
  5. 이동 동선 계산
  6. 일정표 작성
  7. 예산 정리

2. AI 에이전트를 구성하는 핵심 요소

① 목표 인식

"보고서 작성해줘"라는 요청을 받으면 주제, 독자, 필요 자료, 형식, 근거 등을 구체적으로 파악합니다.

② 계획 수립

목표를 작은 작업 단위로 나눕니다. (시장 규모 조사 → 트렌드 정리 → 사례 수집 → 구조 작성 → 다듬기)

③ 도구 사용

상황에 맞는 도구를 골라 작업합니다. (웹 검색, 캘린더, 이메일, 문서 도구, 코드 실행, 이미지 생성 등)

④ 실행과 관찰

작업을 실행한 뒤 결과가 충분한지, 출처가 믿을만한지 등을 판단합니다.

⑤ 반복 개선

한 번에 끝내지 않고 부족한 점을 확인하여 결과물을 더 좋게 다듬는 반복 과정을 거칩니다.

3. 일반 AI vs AI 에이전트

구분 일반 AI 챗봇 AI 에이전트
역할질문에 답변목표를 수행
방식즉시 응답 중심계획 → 실행 → 검토
도구 사용제한적이거나 없음적극적으로 사용
작업 범위단일 답변여러 단계 작업
예시"설명해줘""자료 찾아서 보고서 만들어줘"

일반 AI = 똑똑한 답변자 | AI 에이전트 = 일을 처리하는 디지털 작업자

4. AI 에이전트 활용 예시

여행 에이전트

"3박 4일 부산 여행 계획 짜줘"

  • 날짜/날씨 확인
  • 숙소 위치 고려
  • 이동 동선 및 일정표 작성
업무 자동화 에이전트

"회의록 정리해서 메일 보내줘"

  • 핵심 내용 요약 / 할 일 추출
  • 메일 초안 작성 및 발송
쇼핑 에이전트

"10만 원 이하 무선 키보드 추천해줘"

  • 판매 제품 검색
  • 가격 비교
  • 리뷰/장단점 정리

AI 에이전트는 목표를 이해하고, 작업을 나누고, 도구를 사용하며, 반복 개선하여 최종 결과를 만들어내는 시스템입니다.
